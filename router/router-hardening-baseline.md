# Router Hardening Baseline

## Summary
This document records the initial hardening steps completed on the newer router before deeper network integration.

## Completed Actions
- changed the default admin password
- changed naming so router-identifying information is not exposed
- reset credentials as part of the initial hardening process
- updated the firmware
- began planning SSID naming structure

## Current Approach
More advanced wireless and segmentation-related configuration is being delayed until the switch and firewall are fully in place.

## Reason for This Approach
Holding off on deeper configuration should:
- reduce rework
- make the final setup cleaner
- keep the current environment easier to troubleshoot

## Why This Matters
Even before full firewall deployment, basic router hardening improves the security baseline of the environment and reduces unnecessary exposure.

## Future Follow-Up
- finalize SSID naming plan
- integrate router settings into the broader network design
- align router configuration with final firewall and switch deployment
